## 1. VDI 개요

### VDI란 무엇인가?

**VDI (Virtual Desktop Infrastructure)** 는 사용자 데스크톱 환경을 중앙 인프라(데이터센터 또는 클라우드)에서 실행하고, 사용자는 네트워크를 통해 원격으로 접속하는 방식입니다.

### 핵심 특징

- **중앙 집중식 관리**  
  데스크톱 환경과 정책을 중앙에서 통합 관리

- **다양한 단말 접속 지원**  
  PC, 노트북, 태블릿 등 여러 기기에서 동일한 업무 환경 제공

- **보안 및 통제 강화**  
  데이터와 정책을 중앙 인프라 기준으로 관리 가능

- **유연한 원격 업무 환경**  
  장소와 단말에 덜 종속적인 접속 방식 제공

### VDI vs 전통 PC

| 구분 | 전통 PC | VDI |
|---|---|---|
| 실행 위치 | 로컬 단말 | 중앙 서버 |
| 접속 방식 | 로컬 실행 | 네트워크 원격 접속 |
| 데이터 관리 | 단말 중심 | 중앙 관리 중심 |
| 관리 방식 | 개별 관리 | 중앙 통합 관리 |
| 보안 통제 | 단말별 통제 | 중앙 정책 기반 통제 |

### 왜 VDI를 사용하는가?

#### 1) 보안 및 통제 강화
- 데이터와 정책을 중앙 인프라 기준으로 관리하여 정보 유출 위험 감소
- 인증, 감사, 접근 통제를 일관되게 적용 가능

#### 2) 중앙집중형 운영 효율화
- 패치, 배포, 계정/정책 관리를 중앙에서 수행
- 운영 복잡도 감소 및 관리 효율 증가

#### 3) 원격근무 및 하이브리드 워크 지원
- 사용자는 장소에 구애받지 않고 동일한 업무 환경에 접속 가능

#### 4) 다양한 단말 활용 지원
- 노트북, 태블릿, 브라우저 등 다양한 단말에서 동일한 데스크톱 환경 이용 가능

### 대표 VDI / DaaS 솔루션

#### On-Prem / Hybrid

- Gabia

#### Cloud (DaaS)

### 실제 활용 사례

- **우정사업본부**: 공공 DaaS 도입 확대 사례
- **티디지 & Azure Virtual Desktop**: 국내 민간 서비스형 제공 사례

## 2. VDI 구조와 동작 원리

### VDI 공통 아키텍처

VDI는 일반적으로 다음과 같은 계층으로 구성됩니다.

<img width="354" height="344" alt="image" src="https://github.com/user-attachments/assets/4354027d-5dcb-4691-ad3f-7547e61109cf" />


1. **인증 / 디렉터리 레이어**  
   사용자 로그인, 권한 확인, 접근 제어 담당

2. **연결 브로커 (Connection Broker)**  
   사용자를 적절한 데스크톱 또는 세션에 연결하고 세션 상태 관리

3. **가상화 / 세션 호스트 레이어**  
   가상 데스크톱 또는 사용자 세션이 실제로 실행되는 계층

4. **원격 디스플레이 프로토콜**  
   화면, 입력, 오디오를 네트워크를 통해 사용자 단말에 전달

5. **프로파일 / 스토리지 레이어**  
   VM 이미지, 사용자 프로파일, 데이터 저장 담당

6. **네트워크 인프라**  
   사용자 단말과 서비스 간 안정적인 연결 제공

### VDI 동작 흐름 - 사용자 관점

<img width="485" height="263" alt="image" src="https://github.com/user-attachments/assets/a0267e6a-61cd-4dbc-8b7e-1d9d2e5809f7" />


1. **클라이언트 접속**  
   VDI 앱 또는 웹 브라우저 실행

2. **사용자 인증**  
   계정, 비밀번호, MFA 등으로 로그인

3. **리소스 할당**  
   권한 및 정책에 따라 데스크톱/세션 연결 대상 결정

4. **원격 세션 연결**  
   PCoIP, HDX, RDP 등의 프로토콜을 통해 화면 스트리밍

5. **업무 수행**  
   서버 측 데스크톱 환경에서 애플리케이션 사용

6. **세션 종료**  
   로그아웃 후 세션 유지 또는 초기화

> 핵심 포인트: 사용자의 입력은 서버 측 세션으로 전달되고, 화면만 단말에 표시되므로 데이터는 중앙 인프라에 위치합니다.

## 3. VDI 운영 관점 집중 분석

### 운영 방식의 분류

#### 1) 사용자 환경 유지 방식에 따른 분류

| 구분 | Persistent VDI | Non-Persistent VDI |
|---|---|---|
| 개념 | 사용자별 전용 데스크톱, 설정 유지 | 공용 이미지 기반, 세션 종료 시 초기화 |
| 장점 | 개인화 용이, 사용자 경험 우수 | 관리 단순, 자원 효율 높음 |
| 단점 | 관리 및 스토리지 부담 증가 | 개인화 제한 |
| 적합 대상 | 개발자, 디자이너, 고정 업무 사용자 | 콜센터, 교육장, 공용 업무 환경 |

#### 2) 호스팅 방식에 따른 분류

- **온프레미스 VDI**  
  단일 서버 또는 사내 인프라에서 다수 VM 운영

- **클라우드 기반 DaaS**  
  퍼블릭 클라우드 기반 데스크톱을 서비스 형태로 제공

- **하이브리드 VDI**  
  온프레미스와 클라우드를 혼합하여 보안과 확장성 확보

### 대규모 환경 구축 고려사항

500대 이상 규모의 VDI 환경에서는 다음 요소가 중요합니다.

<img width="316" height="266" alt="image" src="https://github.com/user-attachments/assets/05f8fd4a-1938-42a3-9ab8-d2c67ebb1609" />


#### 고가용성 (HA) 설계
- 연결 브로커, 스토리지, 네트워크 구간의 단일 장애점 최소화
- 장애 시 서비스 연속성 확보

#### 자동화 및 오케스트레이션
- 이미지, 패치, 정책, VM 프로비저닝 자동화
- 운영 복잡도와 인력 부담 완화

#### 모니터링 및 알림 체계
- 세션 호스트, 스토리지, 네트워크 상태를 지속적으로 감시
- 이상 징후를 조기에 탐지

#### 스토리지 / 네트워크 병목 관리
- 프로파일 로딩, IOPS, 지연시간, 대역폭 관리
- 사용자 체감 성능 저하 예방

### 운영자의 핵심 관리 포인트

#### 1) 이미지 관리 (Golden Image)
- 마스터 이미지 생성 및 패치 반영
- 표준화된 데스크톱 환경 제공

#### 2) 리소스 모니터링
- CPU, 메모리, 스토리지, 네트워크 사용량 모니터링
- 병목 구간 파악

#### 3) 정책 관리
- 접근 권한, 보안 정책, 디바이스 제어 정책을 중앙에서 일괄 적용

#### 4) 용량 계획 (Capacity Planning)
- 동시 접속 수와 리소스 사용량을 바탕으로 증설 시점과 규모 예측

### 운영 이슈 사례와 대응

**원인**
- 세션 호스트 자원 경합
- 스토리지 병목
- 프로파일 로딩 지연
- 네트워크 지연

**대응**
- 부하 분산
- 자원 증설

- 네트워크 QoS 적용

## 5. 결론

### 핵심 요약

- VDI는 사용자 단말이 아닌 **중앙 인프라에서 데스크톱을 실행하고 관리하는 기술**입니다.
- 이를 통해 **보안, 운영 효율, 유연성**을 동시에 확보할 수 있습니다.
- 표준화된 업무 환경 제공과 원격근무 지원을 함께 실현할 수 있습니다.

### 미래 전망

- **AI 기반 운영 최적화 확대**
- **DaaS와 하이브리드 업무 환경 확산**
- **제로 트러스트 기반 보안 통합 강화**

> VDI는 단순한 원격 접속 기술이 아니라, 안정적이고 유연한 업무 환경을 구현하기 위한 인프라 기술입니다.

또한 실제 서비스 품질은 기술 자체보다도 **운영 체계**에 크게 좌우됩니다.  
가용성, 모니터링, 병목 관리, 장애 대응 체계를 얼마나 잘 설계하고 운영하느냐가 핵심입니다.
